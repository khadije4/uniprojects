VCS is your solution

VCS (Version Control System ) is a software that helps software developers to work together and maintain a complete history of their work.

Git is the most popular Distributed Version Control System due to its speed, flexibility, and collaboration features.

Git is a distributed version control system that tracks versions of files. It is often used to control source code by programmers who are developing software collaboratively. It was created by Linus Torvalds in 2005, and has been maintained by Junio Hamano since then.

It is used for:
Tracking code changes
Tracking who made changes
Coding collaboration

Git is commonly paired with platforms like GitHub, GitLab, and Bitbucket, which provide cloud-based repositories for enhanced collaboration.


The most popular platform for hosting Git repositories is GitHub.

what is github

GitHub is a proprietary developer platform that allows developers to create, store, manage, and share their code. It uses Git to provide distributed version control and GitHub itself provides access control, bug tracking, software feature requests, task management, continuous integration, and wikis for every project. Headquartered in California, it has been a subsidiary of Microsoft since 2018.

Key Features of GitHub:

  1. Git Repository Hosting
  2. Collaboration Tools
  3. Issue Tracking
  4. GitHub Actions
  5. Version Control
  6. Community and Open Source

how to use git

Using Git involves several steps, from setting it up to managing your code repositories. Here's a simple guide to get started with Git:

  1. install git on linux
    • Dedian/Ubuntu :
      $sudo apt update 
$sudo apt install git
    • fedora :
      $sudo dnf install git

    check if git is properly installed 

    $git --version

2.configure Git

set up your Git username and eamil:

$git config --global user.name "your name"
$git config --global user.email "your email"

view your configuration:

$git config --list

3.create a new Repository

1.open a terminal and navigate to your project directory:

$cd /path/to/your/project

2.initialize Git in the directory

$git init

or copy(clone)an existing remote repository to your local system: 

$git clone https://github.com/username/repository.git

4.add files to staging

1.check the current status :

$git status

2.add files to the staging area :

$git add filename

add all files:

$git add .

5.commit changes

save your changes with a descriptive message:

$git commit -m "intial commit"

6.create and switch branches

1. create a new branch:

$git branch feature_branch

2.switch to the branch:

$git checkout feature_branch

7.connect to a remote repository

1.add a remote repository(github):

$git remote add origin https://github.com/username/repository.git

2.push changes to the remote :

$git push -u origin main

8.pull changes from the remote repository

fetch and integrate changes:

$git pull origin main

9.view commit history

$git log
